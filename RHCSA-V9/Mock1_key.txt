🛠️ Section A Solutions (servera)
Task 1: Password Break Mechanics
Force reboot servera from your hypervisor controller interface.

At the GRUB boot menu selection screen, press the e key to halt execution and enter edit mode.

Locate the text block line initializing the Linux kernel parameters (look for the line beginning with linux or linux16).

Move your cursor to the trailing end of that line, append a space, and write: rd.break

Press Ctrl + x to continue booting into an isolated RAM disk single-user environment.

Execute the following sequence of commands sequentially to unlock and re-label the filesystem:

Bash


mount -o remount,rw /sysroot
chroot /sysroot
echo "RedHatExam2026" | passwd --stdin root
touch /.autorelabel
exit
exit

🛠️ Section C Solutions (serverc)
Task 9: LVM and Swap Real Estate
Bash


# Verify raw disk naming (e.g., /dev/vdb)
pvcreate /dev/vdb
vgcreate -s 32M vg_data /dev/vdb
lvcreate -l 60 -n lv_apps vg_data
mkfs.xfs /dev/vg_data/lv_apps
mkdir -p /apps

# Gather block system UUID
blkid /dev/vg_data/lv_apps

# Append to /etc/fstab file:
# UUID=YOUR-UUID-HERE /apps xfs defaults 0 0
mount -a

# Provision Swap Extent Space
lvcreate -L 1G -n lv_swap_ext vg_data
mkswap /dev/vg_data/lv_swap_ext

# Append to /etc/fstab file:
# /dev/vg_data/lv_swap_ext swap swap defaults 0 0
swapon -a
Task 10: Dynamic File System Expansion
Bash


lvextend -L 3.5G -r /dev/vg_data/lv_apps
(The -r modifier handles resizing the XFS layout natively during block expansion).

Task 12: Autofs Engine Automation
Bash


dnf install -y autofs
mkdir -p /opt/finance_vault

# Set up master mapping file
echo "/data /etc/auto.finance" >> /etc/auto.master

# Construct definition file
echo "finance -fstype=bind :/opt/finance_vault" > /etc/auto.finance

# Enable and cycle services
systemctl enable --now autofs

Task 14: Unprivileged Containerization
Bash


# Execute these commands completely logged in as student on serverc
ssh student@serverc

# Enable user lingering so user services run headless without open SSH lines
sudo loginctl enable-linger student

podman run -d --name web_sandbox -p 8085:8080 registry.access.redhat.com/ubi9/nginx-120

# Construct systemd execution definitions
mkdir -p ~/.config/systemd/user/
cd ~/.config/systemd/user/
podman generate systemd --new --files --name web_sandbox

# Enable user container service
systemctl --user daemon-reload
systemctl --user enable --now container-web_sandbox.service
